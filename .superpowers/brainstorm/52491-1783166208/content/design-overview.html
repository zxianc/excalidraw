Data Flow

Save & Sync Flow
User edits canvas
→ DocumentManager.autoSave() debounce 500ms
→ Write to IndexedDB immediately (offline-safe)
→ Mark document as dirty
→ SyncEngine debounce 2s, then:
  → GET remote version (ETag / version field)
  → if no conflict: PUT local → remote, clear dirty
  → if conflict: show ConflictDialog
    → Option A: Keep local (overwrite remote)
    → Option B: Use remote (overwrite local)
    → Option C: Keep both (save local as copy, load remote)

Config Export/Import Flow

Multi-Device Setup
Device A (setup):
→ User fills: Endpoint, Bucket, AK, SK
→ Settings → "Export Config"
→ Config encrypted with user-chosen password
→ Generates: encrypted string + QR code

Device B (import):
→ Settings → "Import Config"
→ Scan QR code OR paste encrypted string
→ Enter password to decrypt
→ Config applied, sync starts automatically

Implementation Phases

1

Phase 1 - Local Multi-Document + S3 Sync

IndexedDB document store, VSCode sidebar with folders, S3 adapter, conflict detection dialog, config export/import, offline-first save flow. This is the core deliverable.

2

Phase 2 - WebDAV + Polish

WebDAV adapter (NextCloud/Nutstore), document thumbnails, search, drag-and-drop reordering, keyboard shortcuts (Cmd+B toggle sidebar), recent documents list.

3

Phase 3 - Advanced Features

Git adapter, version history UI, batch import/export, document sharing via pre-signed URLs, optional CRDT merge for real-time collaboration.
